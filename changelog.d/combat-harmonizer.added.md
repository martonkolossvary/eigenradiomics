Added `ComBatHarmonizer`, a leakage-safe ComBat batch-harmonization transformer. It recovers ComBat's parameters (grand mean, pooled variance, per-batch empirical-Bayes effects) on the training data via inmoose's estimation helpers and replays the per-batch adjustment on new data, so it composes inside a scikit-learn `Pipeline`/cross-validation (batch labels and numeric covariates flow via `batch=`/`covariates=` keywords and metadata routing). Supports a reference batch, biological-covariate preservation, passthrough/error on unseen batches, and constant-feature passthrough; `fit_transform` reproduces `inmoose.pycombat_norm` exactly. Requires the optional `inmoose` dependency (`combat` extra).
